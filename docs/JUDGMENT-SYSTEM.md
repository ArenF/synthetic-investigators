# CoC 7e 판정 시스템 설계 (검증 완료)

> 작성일: 2026-04-23 | CoC 7th Edition 공식 규칙 기반 검증

---

## 설계 원칙

1. **AI에게는 결과 텍스트만 전달한다.** 판정의 기계적 과정은 프론트+서버가 처리.
2. **GM(인간)이 판정 시점과 유형을 결정한다.** CoC의 근본 구조 유지.
3. **현재의 대화 루프(`send_turn`)는 기반으로 유지한다.** 판정은 대화 흐름 위에 얹히는 이벤트.

---

## 1. 판정 유형 — 4가지

### 1.1 단순 판정 (Simple Check)

가장 기본적인 판정. 게임의 80%+를 차지.

```
입력: 캐릭터 + 기술(값) + 난이도 + [보너스/페널티 다이스]
과정: d100 vs 목표값
결과: 6-tier
```

**목표값 산출:**

| 난이도 | 목표값 | 예시 (기술 60) |
|--------|--------|---------------|
| 일반 (Regular) | 기술값 | 60 |
| 어려움 (Hard) | ⌊기술값 / 2⌋ | 30 |
| 극한 (Extreme) | ⌊기술값 / 5⌋ | 12 |

**6-tier 결과:**

| tier | 조건 |
|------|------|
| 극단적 성공 | roll ≤ ⌊목표값 / 5⌋ |
| 어려운 성공 | roll ≤ ⌊목표값 / 2⌋ |
| 성공 | roll ≤ 목표값 |
| 실패 | roll > 목표값 |
| 나쁜 실패 | roll > ⌊(기술값 + 95) / 2⌋ |
| 대실패 | 기술값 ≥ 50: roll 96~100 / 기술값 < 50: roll 100 |

### 1.2 대항 판정 (Opposed Roll)

두 편이 각자 기술을 굴려서 성공 등급을 비교.

```
입력: A측(캐릭터+기술) + B측(캐릭터 or NPC+기술) + [각 측 보너스/페널티]
과정: 양측 d100 → 각자 성공 등급 산출 → 비교
결과: 승자 결정
```

**비교 규칙:**
- 더 높은 성공 등급이 승리 (극단적 > 어려운 > 일반 > 실패)
- 등급 동일 → 기술값이 높은 쪽 승리
- 기술값도 동일 → 재굴림

**제한:**
- 밀어붙이기 불가
- PC vs NPC 시 대안: NPC 기술값 기반 난이도 변환으로 단순 판정 대체 가능

| NPC 기술값 | PC 판정 난이도 |
|-----------|--------------|
| < 50 | 일반 |
| 50~89 | 어려움 |
| ≥ 90 | 극한 |

### 1.3 결합 판정 (Combined Roll)

하나의 과제에 복수 기술이 필요할 때. **d100을 한 번만 굴려서** 여러 기술값에 대조.

```
입력: 캐릭터 + 기술 목록 + 성공 조건 (AND/OR)
과정: d100 한 번 → 각 기술값과 비교
결과: 조건 충족 여부
```

**예시:** 도서관이용 45, 고고학 50 → roll 48
- 도서관이용: 48 > 45 → 실패
- 고고학: 48 ≤ 50 → 성공
- AND 조건이면 → 실패 (하나 미달)
- OR 조건이면 → 성공 (하나 충족)

### 1.4 그룹 판정 (Group Roll)

복수 인원이 각각 굴림. 결합 판정과 다르게 **인원별로 개별 d100**.

| 변형 | 규칙 |
|------|------|
| 협력 (Cooperative) | 1명이라도 성공 → 전체 성공 |
| 전원 (All-must-succeed) | 전원 성공 필요 (1명 실패 = 전체 실패) |
| 그룹 행운 | 행운이 가장 낮은 캐릭터가 대표 굴림 |

## 2. 수식어 — 2가지

모든 판정 유형에 적용 가능.

### 2.1 난이도

| 난이도 | 목표값 |
|--------|--------|
| 일반 (Regular) | 기술값 |
| 어려움 (Hard) | ⌊기술값 / 2⌋ |
| 극한 (Extreme) | ⌊기술값 / 5⌋ |

### 2.2 보너스/페널티 다이스

d100 굴림 시 텐다이스(십의 자리)를 추가로 굴려서 유/불리 반영.

```
보너스: 텐다이스 중 낮은 것 채택 (유리)
페널티: 텐다이스 중 높은 것 채택 (불리)

예) 일의 자리 = 3, 텐다이스 = 40, 70
    보너스 → 43 (낮은 40 채택)
    페널티 → 73 (높은 70 채택)
```

**규칙:**
- 보너스 1개 + 페널티 1개 = 상쇄 (일반 굴림)
- 최대 2개까지. 3개째가 필요하면 → 난이도를 한 단계 올림

## 3. 판정 실패 후 선택지 — 양자택일

판정 실패 시 GM/플레이어가 선택할 수 있는 후속 경로. **둘 중 하나만 가능.**

### 3.1 밀어붙이기 (Pushing)

**동일 난이도**로 1회 재굴림. 실패 시 GM이 서술적 대가를 부여 (공포 증폭 목적).

```
실패 → 밀어붙이기 선택 → 동일 난이도 재굴림
                            ├── 성공 → 결과 적용
                            └── 실패 → GM이 중대한 대가 서술
```

**불가 상황:**
- 전투 굴림
- SAN 체크
- 대항 판정
- 캐릭터가 실패를 인지하지 못하는 경우

### 3.2 행운 소비 (Luck Spend)

행운 포인트 1점당 주사위 결과를 1 감소. 즉시 성공 전환 가능.

```
roll 57, 목표값 55 → 행운 2점 소비 → roll을 55로 간주 → 성공
```

**불가 상황:**
- SAN 체크
- 행운 굴림
- 대실패 (fumble)

**추가 규칙:**
- 행운으로 성공 전환 시 기술 성장 체크 미부여

## 4. 특수 판정 시퀀스

기본 판정 유형 + 수식어를 조합한 복합 흐름.

### 4.1 SAN 체크

```
트리거: 공포스러운 대상/사건 조우

단순 판정: d100 vs 현재 SAN값 (난이도: 일반)
  ├── 성공 → 소량 SAN 손실 (또는 0)
  └── 실패 → 대량 SAN 손실

손실량 확인:
  ├── 손실 < 5 → SAN 차감만
  └── 손실 ≥ 5 → INT 굴림 필요
       ├── INT 성공 → 일시적 광기 발동 (1D10 시간)
       │              광기 테이블 1D10 굴림
       └── INT 실패 → 공포 억압 (즉시 광기 없음, 잠재적 영향)

무기한 광기 체크:
  하루 누적 SAN 손실 ≥ ⌊현재SAN / 5⌋ → 무기한 광기
  무기한 광기 상태에서 추가 SAN 1점 손실만으로도 광기 재발
```

**밀어붙이기 불가, 행운 소비 불가**

### 4.2 전투 라운드

```
행동 순서: DEX 내림차순 (준비된 화기 = DEX + 50)

공격자 턴:
  단순 판정 (격투/사격 기술)
    ├── 실패 → 빗나감, 턴 종료
    └── 성공 → 방어자 반응 선택

방어자 선택:
  ┌── 회피 (Dodge)
  │   대항 판정: 공격자 vs 방어자(회피 기술)
  │   동점 = 회피 승 (방어 유리)
  │   회피 성공 → 피해 없음
  │   회피 실패 → 피해 적용
  │
  └── 반격 (Fight Back)
      대항 판정: 공격자 vs 방어자(격투 기술)
      동점 = 공격자 승 (공격 유리)
      반격 성공 → 방어자가 공격자에게 피해
      반격 실패 → 공격자 피해 적용

피해 산출:
  무기 피해 다이스 + 피해 보너스(db)
  극단적 성공: 최대 피해 + 추가 피해 보너스 다이스
```

**밀어붙이기 불가** (전투 굴림)
**행운 소비 가능** (전투 굴림에는 적용됨)

### 4.3 추격전

```
설정:
  선형 로케이션 트랙 (위치 1, 2, 3, ...)
  참가자별 MOV 값 확인

행동 수:
  최저 MOV 캐릭터 = 1회 행동
  다른 캐릭터 = 1 + (자기 MOV - 최저 MOV) 회 행동

이동:
  1칸 이동 = 무조건
  2~3칸 이동 = 페널티 다이스 1개
  4~5칸 이동 = 페널티 다이스 2개

장애물/위험:
  그룹 행운 굴림으로 발생 여부 결정
  ├── 행운 성공 → 플레이어 측이 위험 배치 결정
  └── 행운 실패 → GM이 위험 배치
  위험 통과: 해당 기술 단순 판정
  실패 시: 위치 후퇴 또는 부상

차량 추격:
  운전 판정 → 성공도에 따라 MOV 보정
  극단적 성공: MOV +1 / 실패: MOV -1
```

## 5. 현재 시스템과의 갭

### 구현됨 ✓

| 항목 | 위치 |
|------|------|
| 단순 판정 (6-tier) | `dice.ts:48-82` |
| 난이도 (일반/어려움/극한) | `dice.ts:56-58` |
| 대실패 기준 (기술 50 분기) | `dice.ts:64` |
| Effect 적용 (stat/item/status) | `judgment.ts:75-84`, `state.ts` |
| SAN 자동 광기 (일부) | `state.ts:applySanLoss()` |

### 미구현 ✗

| 항목 | 설명 |
|------|------|
| 대항 판정 | 성공 등급 비교 로직 없음 |
| 결합 판정 | 1회 굴림 복수 기술 대조 없음 |
| 그룹 판정 | 협력/전원 변형 없음 |
| 보너스/페널티 다이스 | 텐다이스 추가 굴림 없음 |
| 밀어붙이기 | 재굴림 경로 없음 |
| 행운 소비 | 판정 후 행운 차감 없음 |
| SAN INT 굴림 | 5+ 손실 시 INT 판정 누락 (자동 광기 처리) |
| SAN 무기한 기준 | `max` 기준 → `현재SAN` 기준으로 수정 필요 |
| 전투 대항 처리 | 회피/반격 대항 + 동점 분기 없음 |
| 전투 DEX+50 화기 | 화기 준비 시 순서 보정 없음 |
| 추격 트랙 | 로케이션 기반 시스템 없음 |
| 추격 MOV 행동 수 | MOV 차이 → 행동 수 변환 없음 |

### 수정 필요 ⚠

| 항목 | 현재 | 올바른 값 |
|------|------|----------|
| 무기한 광기 기준 | `floor(san.max / 5)` | `floor(현재SAN / 5)` |

## 6. 판정 플로우 — 프론트 관점

AI에게는 최종 결과 텍스트만 전달. 아래 전체 과정은 프론트+서버가 처리.

### 단순 판정 전체 플로우

```
GM: 캐릭터 + 기술 + 난이도 + [보너스/페널티] 설정
                │
                ▼
           d100 굴림 → 6-tier 결과
                │
           ┌────┴────┐
         성공       실패
           │         │
     결과→AI    대실패인가?
                     │
                ┌────┴────┐
              대실패     일반 실패
                │         │
          대가→AI    밀어붙이기 or 행운소비?
                        (양자택일)
                           │
                  ┌────────┼────────┐
               밀어붙이기   행운소비    수락
                  │         │         │
             동일 난이도    1:1 차감  실패→AI
             재굴림 (1회)   즉시 성공
                  │         │
             ┌────┴────┐  결과→AI
           성공       실패
             │         │
         결과→AI   GM 대가→AI
```

### 대항 판정 전체 플로우

```
GM: A측(캐릭터+기술) + B측(캐릭터/NPC+기술) + [보너스/페널티]
                │
                ▼
        양측 d100 → 각자 성공 등급 산출
                │
                ▼
         등급 비교
           │
      ┌────┼────┐
    A 승  동점  B 승
      │    │     │
      │  기술값   │
      │  비교     │
      │    │     │
      │  높은 쪽  │
      │  승리     │
      │  (동일→   │
      │  재굴림)  │
      │          │
      ▼          ▼
   결과→AI    결과→AI
```

### SAN 체크 전체 플로우

```
GM: 대상 캐릭터 + 성공 시 손실 + 실패 시 손실
                │
                ▼
         d100 vs 현재 SAN
                │
           ┌────┴────┐
         성공       실패
           │         │
      소량 손실   대량 손실
           │         │
           └────┬────┘
                │
         손실량 < 5?
           │
      ┌────┴────┐
    예(<5)     아니오(≥5)
      │         │
   SAN 차감   INT 굴림
   → AI        │
          ┌────┴────┐
        INT 성공   INT 실패
          │         │
     일시적 광기   억압
     1D10시간      (SAN 차감만)
     광기테이블    → AI
     → AI
                │
         무기한 광기 체크:
         하루 누적 ≥ ⌊현재SAN/5⌋?
           │
      ┌────┴────┐
     예         아니오
      │          │
   무기한 광기  → AI
   → AI
```
