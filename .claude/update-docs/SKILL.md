## 판별 기준

**기록해야 하는 것** (미래 Claude가 코드만 봐서는 알기 어려운 것):
- 새로운 아키텍처 패턴이나 설계 결정 (왜 이렇게 했는지)
- 트릭/제약/주의사항 (예: Extended Thinking에서 temperature 제거 필수)
- 새로운/변경된 WebSocket 메시지 타입 또는 프로토콜
- 새로운/변경된 핵심 타입 인터페이스
- 알려진 버그 또는 의도적인 미구현 항목
- 데이터 파일 스키마 변경
- 개발 워크플로우 변경

**기록하지 않아도 되는 것** (코드에서 바로 읽히는 것):
- 단순 버그 수정 (패턴 변화 없음)
- 기존 패턴을 따른 단순 기능 추가
- 스타일/포맷 변경
- 코드를 읽으면 바로 알 수 있는 구현 세부사항

## 실행 절차
1. 이 대화에서 변경된 코드/파일을 파악한다.
2. 위 기준으로 CLAUDE.md 기록 가치를 판별한다.
3. `.claude/CLAUDE.md`를 읽는다.
4. 기록할 내용이 있으면:
   - 기존 관련 섹션을 찾아 **해당 부분만** Edit으로 수정한다 (전체 재작성 금지).
   - 해당하는 섹션이 없으면 가장 적절한 위치에 새 항목을 추가한다.
   - 이미 기록된 내용과 중복되지 않도록 확인한다.
5. 결과를 보고한다:
   - 업데이트했다면: 어떤 섹션의 무엇을 추가/수정했는지
   - 업데이트 불필요하다면: 그 이유 (단순 버그 수정, 코드에서 자명, 이미 기록됨 등)
